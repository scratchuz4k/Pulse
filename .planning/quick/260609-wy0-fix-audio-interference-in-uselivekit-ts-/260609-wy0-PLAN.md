<objective>
Fix double audio playback in the LiveKit composable. The current `TrackSubscribed` handler manually builds an `<audio>` element from `track.mediaStreamTrack`, while the LiveKit SDK already manages playback internally — producing duplicated/echoing remote audio. Replace manual attachment with the SDK's `track.attach()` / `track.detach()` lifecycle.

Purpose: Eliminate audio interference; align with LiveKit's intended API.
Output: Updated `useLiveKit.ts`.
</objective>

  <name>Task 1: Replace manual audio element with track.attach() / track.detach()</name>
  <files>pulse-client/src/renderer/composables/useLiveKit.ts</files>
  <action>
In the `RoomEvent.TrackSubscribed` handler (currently lines 59-75): remove the manual element construction (`track.mediaStreamTrack`, `document.createElement('audio')`, `srcObject = new MediaStream([raw])`, `appendChild`, `el.play()`). For audio tracks call `const el = track.attach()` — this returns an `HTMLMediaElement` the SDK manages. Set `el.id = \`livekit-audio-${participant.identity}\``, append it to `document.body`, and if `activeOutputId.value` is set and `el.setSinkId` is a function call `el.setSinkId(activeOutputId.value).catch(() => {})`. Keep the existing console.log.

In the `RoomEvent.TrackUnsubscribed` handler: replace the `document.getElementById(...)?.remove()` line with `track.detach().forEach((el) => el.remove())` (rename the `_track` param to `track`). This both stops SDK playback and removes the DOM elements.

Leave the `ParticipantDisconnected` handler's `document.getElementById(\`livekit-audio-${participant.identity}\`)?.remove()` as a defensive cleanup (TrackUnsubscribed fires first via detach; the id-based remove is a harmless backstop).

`switchOutput` already queries `audio[id^="livekit-audio-"]` — the preserved `el.id` keeps it working, so do not change `switchOutput`.
  </action>

  <action>
`connect()` calls `setMicrophoneEnabled(true)` (mic already live) and then `await refreshDevices()`, which opens a second `getUserMedia` stream purely to unlock device labels. Since the LiveKit mic is already active at that point, device labels are already unlocked. Add an optional parameter `refreshDevices(unlockLabels: boolean = true)`. When `unlockLabels` is false, skip the `navigator.mediaDevices.getUserMedia(...)` block entirely and go straight to `enumerateDevices()`. In `connect()`, call `await refreshDevices(false)` (mic already provides label permission). Keep the default `true` behavior for any other caller.
