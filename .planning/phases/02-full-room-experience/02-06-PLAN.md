<task type="auto" tdd="false">
  <name>Task 2: Replace globalShortcut with uiohook-napi in main process</name>
  <files>pulse-client/src/main/index.ts</files>
  <behavior>
    - When PTT key is pressed (any focus state), main sends `ptt:key-down` once (debounced — uiohook repeats keydown while held).
    - When PTT key is released, main sends `ptt:key-up`.
    - Other apps still receive the key (no exclusive capture).
  </behavior>
  <action>
Remove all `globalShortcut` usage (import, `globalShortcut.register/unregister/unregisterAll`). Import `{ uIOhook, UiohookKey }` from `uiohook-napi`.

Build a mapping from the stored accelerator string (e.g. 'A', 'Space', 'F1') to the uiohook keycode. uiohook provides `UiohookKey` (e.g. `UiohookKey.Space`, `UiohookKey.A`, `UiohookKey.F1`). Create a helper `acceleratorToUiohookKey(accelerator): number | null` covering: single letters A-Z, digits 0-9, F1-F12, Space, Return/Enter, Backspace, Tab, Escape, Up/Down/Left/Right. This mirrors the renderer `codeToAccelerator` output set.

Register `uIOhook.on('keydown', e => ...)` and `uIOhook.on('keyup', e => ...)`. In keydown: if `e.keycode === currentPttKeycode` and not already held, set `pttHeld = true` and `mainWindow.webContents.send('ptt:key-down')`. uiohook fires keydown repeatedly while held — guard with the `pttHeld` flag so only the first sends. In keyup: if `e.keycode === currentPttKeycode` and `pttHeld`, set `pttHeld = false` and send `ptt:key-up`.

Keep a module-level `currentPttKeycode: number | null`. `ptt:set-key` validates the accelerator is a non-empty string, maps it to a keycode (return false if unmappable), stores `store.set('ptt.key', accelerator)`, and updates `currentPttKeycode`. On startup, load `store.get('ptt.key')` and set `currentPttKeycode`.

Call `uIOhook.start()` once after the window is created. Call `uIOhook.stop()` in the `window-all-closed` handler (replacing `globalShortcut.unregisterAll()`).

Keep the existing `mainWindow.on('blur')` release fallback — but since uiohook now delivers real keyup, the blur handler only matters if a keyup is missed; leave it sending `ptt:key-up` guarded by `pttHeld`.
  </action>
