# Plan 03-03: Admin UI + Priority Speaker Indicator

## Goal
Admin sees assign/remove controls per participant. All participants see a priority speaker badge. Wire admin actions to SignalR hub methods.

### Task 2 — Add `isAdmin` computed to room store
**File:** `pulse-client/src/renderer/stores/room.ts`

The room store needs to know the current room's `createdByUserId`. The rooms list already has this from the API. Add:
```ts
// Computed: current user is admin of the active room
const isAdmin = computed(() => {
  if (!currentRoom.value) return false
  const room = roomList.value.find(r => r.name === currentRoom.value)
  return room?.createdByUserId === authStore.userId
})
```

Export `isAdmin` from the store.

Note: `authStore.userId` must exist — check auth store and add if missing.

### Task 3 — Expose `assignPrioritySpeaker` / `removePrioritySpeaker` from `usePresence`
**File:** `pulse-client/src/renderer/composables/usePresence.ts`

Add two new methods to the returned object:
```ts
async function assignPrioritySpeaker(roomName: string, userId: string): Promise<void> {
  if (!hubConnection) return
  await hubConnection.invoke('AssignPrioritySpeaker', roomName, userId)
}

async function removePrioritySpeaker(roomName: string): Promise<void> {
  if (!hubConnection) return
  await hubConnection.invoke('RemovePrioritySpeaker', roomName)
}
```

Add `PrioritySpeakerChanged` handler alongside the other `hubConnection.on(...)` registrations:
```ts
hubConnection.on('PrioritySpeakerChanged', (userId: string | null) => {
  const { setPrioritySpeaker } = useLiveKit()
  setPrioritySpeaker(userId)
  roomStore.setPrioritySpeaker(userId)
})
```

### Task 4 — Add priority speaker badge + admin controls to participant list in RoomView
**File:** `pulse-client/src/renderer/views/RoomView.vue`

In the participant list template, for each participant entry:
1. Show a microphone/star icon badge if `participant.userId === roomStore.prioritySpeakerId`
2. If `roomStore.isAdmin`, show a context button:
   - "Assign priority speaker" if participant is not the current priority speaker
   - "Remove priority speaker" if participant IS the current priority speaker

Example structure (adapt to existing participant list markup):
```vue
<div v-for="p in roomStore.participants" :key="p.connectionId" class="participant-row">
  <span class="participant-name">{{ p.displayName }}</span>
  <span v-if="p.userId === roomStore.prioritySpeakerId" class="ps-badge" title="Priority Speaker">★</span>
  <button
    v-if="roomStore.isAdmin"
    class="ps-btn"
    @click="togglePrioritySpeaker(p.userId)"
  >
    {{ p.userId === roomStore.prioritySpeakerId ? 'Remove PS' : 'Assign PS' }}
  </button>
</div>
```

Wire the click handler:
```ts
function togglePrioritySpeaker(userId: string) {
  if (roomStore.prioritySpeakerId === userId) {
    presence.removePrioritySpeaker(roomStore.currentRoom!)
  } else {
    presence.assignPrioritySpeaker(roomStore.currentRoom!, userId)
  }
}
```
