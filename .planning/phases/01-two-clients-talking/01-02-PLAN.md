# Plan 01-02: Electron+Vue Client Shell

## Objective

Scaffold the Pulse Electron+Vue 3 desktop client: project setup, auth flow, secure token storage, SignalR presence connection, and LiveKit token fetching. By end of this plan the client can authenticate and retrieve a LiveKit room token. LiveKit audio is wired in Plan 01-03.

## Context

- Framework: Electron + Vue 3 + Vite + TypeScript (`electron-vite/electron-vite-vue` template)
- Presence: `@microsoft/signalr` (Pulse room membership events only — no WebRTC signaling)
- LiveKit audio: `livekit-client` npm package (installed here, used in Plan 01-03)
- Token storage: `electron-store` with encryption (key from `node-machine-id`)
- Server base URL: `http://localhost:5000` (dev), from env var

### Task 3: Preload bridge for secure token storage

<action>
Update `pulse-client/src/preload/index.ts`:
```typescript
import { contextBridge, ipcRenderer } from 'electron'

contextBridge.exposeInMainWorld('pulseApi', {
  storeGet: (key: string) => ipcRenderer.invoke('store:get', key),
  storeSet: (key: string, value: string) => ipcRenderer.invoke('store:set', key, value),
  storeDel: (key: string) => ipcRenderer.invoke('store:del', key),
})
```

Update `pulse-client/src/main/index.ts`:
- Import `electron-store` and `node-machine-id`
- Create `Store` instance with `encryptionKey: await machineId()`
- Register `ipcMain.handle` for `store:get`, `store:set`, `store:del`

Add `window.pulseApi` TypeScript declaration to `pulse-client/src/renderer/env.d.ts`.
</action>

<acceptance_criteria>
- Preload uses `contextBridge.exposeInMainWorld` (not `require`)
- Main process handles the three IPC channels
- `electron-store` uses machine-ID-derived encryption key
- TypeScript knows about `window.pulseApi`
</acceptance_criteria>

```typescript

// login(displayName, password): POST /auth/login → setTokens → persist
```
