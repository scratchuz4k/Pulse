# Plan 01-03: LiveKit Voice Connection

## Objective

Wire LiveKit audio on top of the already-working auth and presence layer. After this plan, any number of clients in the same Pulse room will hear each other's microphone audio routed through the LiveKit SFU. No manual RTCPeerConnection code — `livekit-client` handles the full WebRTC lifecycle.

## Context

- Depends on Plan 01-01 (C# server issues LiveKit tokens at `POST /rooms/token`)
- Depends on Plan 01-02 (client has `fetchLiveKitToken`, `usePresence`, room store)
- LiveKit dev server must be running: `livekit-server --dev` (default port 7880)
- `livekit-client` npm package already installed in Plan 01-02

## Prerequisites (before running this plan)

The executor must verify LiveKit server is available:
```bash
# Install LiveKit CLI if not present
# Windows: winget install LiveKit.LiveKit  OR  download from https://github.com/livekit/livekit/releases
```

### Task 3: Wire LiveKit room events

<read_first>
- `pulse-client/src/renderer/composables/useLiveKit.ts` (from Task 2)
- `pulse-client/src/renderer/stores/room.ts`
</read_first>

<action>
Inside the `connect` function, register these `RoomEvent` handlers before calling `room.connect()`:

```typescript
room.on(RoomEvent.TrackSubscribed, (track, publication, participant) => {
  if (track.kind === Track.Kind.Audio) {
    // Attach audio track to a new <audio> element
    const el = track.attach()
    el.id = `livekit-audio-${participant.identity}`
    document.body.appendChild(el)
  }
})

room.on(RoomEvent.TrackUnsubscribed, (track, publication, participant) => {
  track.detach().forEach(el => el.remove())
})

room.on(RoomEvent.ActiveSpeakersChanged, (speakers) => {
  activeSpeakers.value = speakers.map(s => s.identity)
})

room.on(RoomEvent.ParticipantDisconnected, (participant) => {
  // Remove audio elements for disconnected participant
  document.getElementById(`livekit-audio-${participant.identity}`)?.remove()
})

room.on(RoomEvent.Disconnected, () => {
  isConnected.value = false
  isMicEnabled.value = false
})
```
</action>

<acceptance_criteria>
- `TrackSubscribed` attaches incoming audio tracks to `<audio>` elements appended to `document.body`
- `TrackUnsubscribed` detaches and removes the audio elements
- `ActiveSpeakersChanged` updates the `activeSpeakers` ref with current speaker identities
- `ParticipantDisconnected` removes orphaned audio elements
- `Disconnected` resets reactive state
</acceptance_criteria>

```typescript
async function joinRoom() {
  // 1. Fetch LiveKit token from C# server
  const { liveKitToken, liveKitHost } = await useAuth().fetchLiveKitToken(roomName.value)
  
  // 2. Connect SignalR presence hub
  await usePresence().connect(serverUrl)
  await usePresence().joinRoom(roomName.value)
  
  // 3. Connect LiveKit audio
  await useLiveKit().connect(liveKitToken, liveKitHost)
}
```

```typescript

  await useLiveKit().disconnect()
```
