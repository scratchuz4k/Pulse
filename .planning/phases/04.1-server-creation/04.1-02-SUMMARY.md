## Summary

Refactored PresenceHub to replace env-var-based admin identity with DB-backed server ownership checks. All broadcasts are now server-scoped via SignalR groups. PULSE_ADMIN_USER_ID and IConfiguration dependency fully removed.

## What Was Built

- `Pulse.Server/Hubs/PresenceHub.cs` — Complete refactor:
  - Removed `IsServerAdmin()` / `PULSE_ADMIN_USER_ID` / `IConfiguration` dependency
  - Added `IsOwnerOfServer(int serverId)` — async DB check: `_db.Servers.AnyAsync(s => s.Id == serverId && s.OwnerId.ToString() == callerUserId)`
  - `OnConnectedAsync`: subscribes connection to `server-{serverId}` SignalR group for all servers user belongs to; removed YouAreAdmin broadcast
  - `BroadcastRoomListAsync(int serverId)`: scoped to `_db.Rooms.Where(r => r.ServerId == serverId)` and broadcasts to `Clients.Group("server-{serverId}")` only
  - `BuildRoomListPayload`: dropped `Guid? CreatedByUserId` from signature
  - `AssignPrioritySpeaker` / `RemovePrioritySpeaker`: use `IsOwnerOfServer(room.ServerId)` instead of `CreatedByUserId` comparison
  - `WhisperGroup` record: added `ServerId` field; `CreateWhisperGroup` accepts `int serverId` as first parameter
  - `AddWhisperMember` / `RemoveWhisperMember` / `DissolveWhisperGroup`: use `IsOwnerOfServer(group.ServerId)`
  - `BuildVisibleGroupsForUser`: now async `Task<List<object>>`; derives admin status from `ownedServerIds.Contains(g.ServerId)` DB query
